# Aleksi Lab v1.7.2 Clean Reset Design

## Purpose

Aleksi Lab v1.7.2 is a structural reset, not another visual patch. It preserves
the existing site content and warm editorial identity while removing the
versioned CSS override stack, duplicated Works data transforms, stale animation
code, malformed content paths, and inconsistent version metadata.

The finished package is
`aleksi-lab-total-handoff-v1.7.2-clean-reset.zip`.

## Approved direction

The supplied “Aleksi Lab v1.7.2 Clean Reset Plan” is the authoritative design
brief. Where it offers a recommendation, this design adopts that recommendation.

For Works:

- Desktop at 1180px and above keeps a spatial exhibition wall.
- Tablet from 768px through 1179px uses a stable two-column grid.
- Mobile below 768px uses a single-column reading flow.
- Hover means “lift”: modest elevation, scale, stronger clay border and shadow,
  image-window shift, image scale, and CTA emphasis.
- Active means “focus”: the selected card becomes the visual stage while peers
  recede. On tablet and mobile, focus is simplified to elevation and emphasis so
  content never overlaps.
- Reduced-motion users receive effectively immediate state changes.

## Architecture

### CSS

`styles.css` becomes an import manifest for a formal CSS system:

```text
assets/css/
  reset.css
  tokens.css
  base.css
  layout.css
  navigation.css
  components.css
  pages/
    home.css
    works.css
    work-detail.css
    math.css
    manuscripts.css
    protocol.css
    atlas.css
  utilities.css
```

The existing `99-*`, `100-*`, `101-*`, and `102-*` files are decomposed and
merged into their owning formal files. `legacy.css` is mined for still-used
rules, then removed from the runtime import graph. No new hotfix file is added.

### Works data

`works-data.js` exports one complete object per work. Post-definition override
maps are removed. Each work owns its stable slug, title, source, media paths,
article status, tags, score data, and card tone.

Incorrect legacy slugs are replaced with source-correct names. Compatibility
aliases may be retained only in a small redirect lookup, never as canonical
data. `hasArticle` explicitly controls whether the CTA says “打开阅读” or
“查看作品”.

Works media is normalized to:

```text
content/design/works/<slug>/
  thumb.webp
  hero.webp
  source.<ext>
  article.md
```

Only assets that can be source-correctly assigned are moved into this structure.
Unreferenced but potentially valuable originals go to `content/archive/assets`;
provably redundant generated or thumbnail assets are removed.

### Works interaction

`works.js` is responsible only for:

- navigation rendering;
- card rendering;
- responsive position calculation;
- hover/focus presentation through CSS;
- active and muted state management;
- keyboard activation;
- detail-page navigation.

The old WAAPI snapshot/FLIP functions are deleted. Desktop card positions are
computed from container dimensions rather than a fixed coordinate table.
State changes are represented by `is-active` and `is-muted` plus a small set of
CSS custom properties.

### Navigation

All pages use one stable anchor structure. Active navigation text stays visible,
warm white, and semibold, with a clay marker. Letter-swap behavior is removed
from navigation. Hover, focus, and active styles never hide or displace labels.

### Work detail

The score section is a dedicated panel with a heading region and a separate
score grid. Score cards have minimum height and internal spacing. At narrow
viewports the grid becomes one column. Heading, numbers, labels, and decorative
rules cannot share overlapping positioning contexts.

### Content indexes

Malformed `#U...` filenames are replaced with stable English slugs. The Math
chapter manifest and internal links are updated before regenerating
`content/markdown-index.json` and `content/content-bundle.js`.

Duplicate cards and notes are classified by purpose:

- `notes/` contains canonical long-form learning notes;
- `cards/` contains genuinely card-specific material;
- identical copies are archived or removed from the public index;
- each public item has one entry.

### Versioning and documentation

Runtime and user-facing version strings are unified as
`v1.7.2-clean-reset`. `package.json` uses semver `1.7.2`.

Root `FIX_NOTES_*` files move to `docs/archive/fix-notes/`. The root keeps
`README.md`, `CHANGELOG.md`, and `CLEAN_AUDIT_v1.7.2.md` as the maintenance
entry points.

`scripts/browser-qa-v1.6.js` becomes `scripts/browser-qa.js`.

## Error handling and resilience

- A missing work image renders a neutral fallback frame without breaking card
  content.
- An unknown work slug falls back to the Works index with a readable status,
  rather than silently rendering an unrelated first work.
- Missing article data always yields “查看作品”; it never creates a broken
  reading link.
- Content generation fails loudly on duplicate public source paths, malformed
  `#U...` names, duplicate work slugs, or missing referenced files.
- Responsive card layout is recalculated from the stage width with a debounced
  resize handler.

## Verification

### Automated

`qa-check.js` is rewritten for v1.7.2 and verifies:

1. no hotfix imports or forbidden new patch filenames;
2. the formal CSS file graph;
3. reduced `!important` use;
4. hover, active, muted, and reduced-motion rules;
5. visible active navigation rules;
6. score-panel spacing and minimum heights;
7. canonical Works slugs and single-definition data;
8. CTA semantics from `hasArticle`;
9. unified version strings;
10. no malformed `#U...` paths in generated indexes;
11. no missing references or obvious unreferenced media;
12. no card content overlays;
13. no dead WAAPI/FLIP helpers.

Required commands:

```text
npm install
npm run qa
npm run build
node scripts/browser-qa.js
```

### Browser

Visual checks cover Works, work detail, Math, Manuscripts, and Protocol at:

- 1440px desktop;
- 1366px desktop;
- 1024px tablet;
- 390px mobile;
- reduced-motion mode.

Works checks include default, hover/focus, active, deactivated, keyboard, and
mobile states. Screenshots are retained as audit evidence.

## Delivery

The cleaned repository is copied into `outputs/` and packaged as
`aleksi-lab-total-handoff-v1.7.2-clean-reset.zip`. The ZIP excludes `.git`,
dependency folders, visual-companion sessions, QA screenshots, and temporary
working files. A second copy is handed off to the user’s Desktop after final
verification.
