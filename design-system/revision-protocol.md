# Revision Protocol Design Note

Public UI should call the system **Revision Protocol / 可迭代修订协议** or **Learning Diagnosis Protocol / 可迭代学习诊断协议**.

The internal uploaded skill is archived under `content/skills/learning-diagnosis-protocol/`, but the public website should not use the internal legacy name as a main navigation label.

## Voice Rules

- Diagnose the learning system, not identity or talent.
- Require one observable output.
- Turn a block into a reusable asset.
- Mark raw material as Inbox or Candidate until checked.

## UI Surfaces

- Protocol: method constitution.
- Math Lab: definition / example / counterexample / proof / check / card cycle.
- Works: every work has critique, score, article, and next revision.
- Atlas: connects notes, works, skills, and protocols.
